# MVP 设计

## 1. 第一阶段目标

第一阶段做一个本地自动化研究闭环，不做实盘自动交易。

交付目标：

```text
每天收盘后自动拉数据
每天自动计算特征
每天自动生成明日候选股
每天自动跑规则表现更新
每天自动生成机械复盘
前端可以查看候选股、规则表现、板块强度和交易计划
```

## 2. 暂不做的事情

第一阶段不做：

- 券商实盘下单
- 高频交易
- 完整分钟级回测
- 大规模机器学习预测
- 多用户权限系统
- 复杂研报爬取

## 3. 第一版技术栈

```text
Backend: FastAPI
Frontend: Next.js
Database: PostgreSQL
Cache/Queue: Redis
Jobs: Celery Beat + Celery Worker
Data: AKShare first, Tushare optional
Analysis: Polars + Pandas
Deploy: Docker Compose
```

## 4. 第一版页面

### 市场总览

- 今日大盘状态
- 市场情绪分
- 涨跌家数
- 涨停/跌停数量
- 强势板块排行

### 板块强度

- 板块 1/3/5/20 日强度
- 成交额放大
- 涨停数量
- 龙头股
- 机械强势原因

### 候选股

- 明日候选股
- 所属板块
- 触发规则
- 买入条件
- 止损
- 止盈
- 仓位建议

### 策略规则

- 规则列表
- 状态
- 近 20/60/120 日表现
- 启用/禁用

### 回测报告

- 收益曲线

- MFE/MAE
- 按市场环境分组

### 复盘报告

- 每日机械复盘
- 板块表现复盘
- 规则表现变化
- 风险提示

## 5. 第一版定时任务

### 08:30 pre_market_check

```text
检查昨日任务状态
检查交易日历
输出今日交易计划摘要
```

### 15:30 sync_daily_market_data

```text
同步指数日线
同步个股日线
同步板块日线
同步涨跌停数据
```

### 16:30 compute_daily_features

```text
计算个股特征
计算板块特征
计算市场状态
```

### 18:00 generate_trade_plans

```text
运行启用规则
生成明日候选股
生成买入、止损、止盈和仓位计划
```

### 21:00 run_rule_regression

```text
更新所有规则的滚动表现

```

```text
生成机械复盘报告
总结市场、板块、规则和候选股变化
```

## 6. 第一批规则

### R001 强势板块放量突破

适合短线。

买入候选：

```text
板块强度分 >= 75
个股相对强度分 >= 70
成交额位于过去 60 日前 20%
收盘价距离 20 日新高 <= 3%
非 ST
非停牌
成交额满足流动性要求
```

买入触发：

```text
T 日生成信号
T+1 突破 T 日最高价后买入
如果高开超过 6%，放弃或等待回踩
```

止损：

```text
min(买入价 - 1.5 * ATR14, 突破 K 线低点)
```

止盈：

```text
最高价回撤 6% 止盈
或 5 个交易日未创新高退出
```

### R002 强势板块缩量回踩

适合短线到波段。

买入候选：

```text
板块强度分 >= 70
个股 20 日趋势向上
回踩 MA10 或 MA20 附近
回踩期间成交缩量
未跌破前一平台
```

止损：

```text
跌破 MA20 或结构低点
```

止盈：

```text
达到前高附近减仓
突破后改移动止盈
```

### R003 弱势市场防守过滤

不是独立买入规则，而是市场过滤器。

```text
如果市场状态为 panic 或 weak_trend:
  短线规则仓位降低 50%
  禁止高开追涨
  只允许强势板块前 5 名
```

## 7. 机械复盘模板

每日机械复盘包含：

```text

强势板块
弱势板块
情绪变化

剔除候选股

明日交易计划摘要
```
